Auswahl.Text Plugins

From Elexis-Wiki
Jump to navigation Jump to search

Elexis erlaubt es, verschiedene Text-Plugins anzubinden. Folgende Rahmenbedingung gelten:

  • Auf einem PC kann jeweils nur eine Anbindung aktiv sein.
  • Ein Text-Plugin unterstützt (auf jeden Fall im Moment) genau eines der folgende Text-Formate
    • OpenDocumentFormat (Hilotec-OpenDocument, OpenOffice von Medelexis, JSigle, Text-Templator für MacOSX)
    • MS-Word

Auswahl

Folgende Text-Anbindungen für Elexis (ab Version 3.4) sind bekannt.

  • Hilotec ODF von Antoine Kaufmann, gewartet von Niklaus Giger
    • OpenSource (EPL)
    • Läuft unter Linux, MacOSX, Windows 32 und 64 bit. Es können beliebige Programme verwendet werden, welche das OpenDocument-Format unterstützen, z.B. LibreOffice, OpenOffice, MS-Word.
    • Öffnet für jedes Dokument ausserhalb von Elexis ein Dokument-Fenster. Dort kann das Dokument bearbeitet werden. Nach dem Schliessen wird es im Elexis abgespeichert.
    • Stabilität: gut
    • Warnung: Kann unter Elexis 3.7 nicht verwendet werden, da dort eine Bibliothek nicht gefunden wird.
  • OpenOffice von Medelexis
    • proprietär. Basiert auf NOA
    • Läuft nur unter Windows.
    • Stabilität scheint unter Java7 (oder grösser) deutlich besser zu sein.
    • Die Dokumente werden in einer Ansicht innerhalb von Elexis geöffnet und bearbeitet.
  • MSword_js von Jörg Sigle
    • OpenSource (GPL). Basiert auf jacob, einer Java-Com Brücke zum Aufruf von COM Automationskomponenten
    • Läuft nur unter Windows 32-bit.
    • Stabilität: Beim einzigen Benutzer (Jürg Hamacher) stürzt es etwa 5 Mal pro Jahr ab. Wahrscheinlich aber jeweils nur weil Outlook ein Problem hatte
    • Offene Probleme: DLL für 64-bit fehlt, Verhalten beim Schliessen von Elexis nicht genügend getestet
  • MS-Word von Medelexis
    • proprietär. Basiert auf Eclipse OleAutomation
    • Kann alle Dokumente (nicht jedoch die Vorlagen) aus dem OpenDocument-Format ins MS-Word-Format umwandeln (via | jodconverter). Eine Ausgabe im OpenDocument-Format ist nicht möglich.
    • Läuft nur unter Windows (32-bit und 64-bit)
    • Drucken von Rechnungen scheint langsam zu sein. Die Dokumente werden in einer Ansicht innerhalb von Elexis geöffnet und bearbeitet.
    • Stabilität ist ein ungelöstes Problem
  • DOCX von Medelexis. Verfügbar ab Elexis 3.7
    • proprietär. Basiert auf docx4j
    • Läuft nur unter Windows, Linux, MacOSX
    • Öffnet für jedes DOCX-Dokument ausserhalb von Elexis ein Dokument-Fenster mit einem Programm, dass diese Format unterstützt (z.B. MS-Office, LibreOffice). Dort kann das Dokument bearbeitet werden. Nach dem Schliessen wird es im Elexis abgespeichert.
    • Stabilität: Gut, Schnell beim Drucken von Rechnungen
  • Text-Templator von Gerry Weirich.
    • OpenSource (EPL). Basiert auf JDOM
    • Läuft unter Linux, MacOSX, Windows 32 und 64 bit. Es können beliebige Programme verwendet werden, welche das OpenDocument-Format unterstützen, z.B. LibreOffice, OpenOffice, MS-Word, Apple(tm) iWork(tm) Pages(tm))
    • Da es sich um Templates handelt, ist das erstmalige Aufsetzen der Vorlagen komplizierter als bei anderen Anwendungen.
    • Stabilität: Keine Probleme bekannt
  • NOAText_jsl von Jörg Sigle
    • OpenSource (GPL). Basiert auf NOA (hier gäbe es unter noa-libre eine Bibliothek, welche neuere LibreOffice-Versionen unterstützt)
    • Läuft nur unter Windows 32-bit.
    • Unterstützt nur einige (veraltete) 3.x-er Versionen von Open/Libre-Office. Die Stabilität ist je nach Libre/OpenOffice-Version sehr verschieden! Es lohnt sich also, die Hinweise auf seiner Seite zu beachten!
    • Dieses Plugin scheint von niemanden mehr verwendet zu werden

Wie wählen?

Die Entscheidung für den Anwender wird durch die Beantwortung folgender Fragen erleichtert.

  • Arbeite ich unter MacOSX? Wenn Ja, falls ich was bewährtes nehmen will, brauche ich den Text-Templator. Falls ich mir das Erstellen der Vorlagen erleichern will, versuche ich es mit dem OpenDocument Format von HiloTec
  • Will ich mit MS-Word arbeiten? Wenn ich bereit bin, ein proprietäres Plug-In zu akzeptieren, brauche ich das MS-Word] von Medelexis. Andernfalls kann ich OpenDocument Format von HiloTec einsetzen.
  • Wer mit dem OpenOffice/LibreOffice arbeitet, kann leicht zwischen den Plugins OpenDocument Format von HiloTec, OpenOffice von Medelexis (proprietär) und NOAtext-Version von JSigle hin und her wechseln.

Konfiguration/Vorlagen

Alle Text-Plug-Ins verwenden Vorlagen, welche die Arbeit erleichtern. Diese Vorlagen können durch die Verwendung von Platzhaltern so angepasst werden, dass z.B. die Daten zum Arzt und aktuellen Patienten automatisch eingefügt werden. Vorlagen findet man unter

  • https://medelexis.ch/downloads/ (Fenster rechts/links für MS-Word, OpenDocument, Pages)
  • https://github.com/hilotec/elexis-vorlagen (*.odt für OpenDocument. Einige Vorlage brauchen die nur vom Hilotec-OpenDocument unterstützen Formattierungen in Tabellen)

Das Vorgehen ist unter Textvorlagen_Management dokumentiert.

Das Benützen von Platzhalter ist unter Doc_de#Text-Platzhalter und http://www.rgw.ch/elexis/dox/PlatzhalterInKontrollfeldern.pdf beschrieben. Für eine Liste aller Platzhalter wählen Sie in Elexis im Menü “Fenster..Ansicht..Other” die Ansicht Platzhalter aus.
