Release 3.4.0 - 25.12.2017

In der neuen Version wurden Fehler seit Elexis 3.1.8 und es gibt diverse neue Feature behoben.

In der untenstehenden Liste beziehen sich die Zahlen in Klammern auf das nicht öffentliche Redmine-Ticket-System der Medelexis AG.

Die Benutzersicht für Elexis OpenSource-Anwender für den Update-Prozess wurde von Gerry Weirich unter http://www.elexis.ch/ungrad/reference/update_34/ ebenfalls dokumentiert. Das meiste finden sie hier unten auch.

Teile, welche nur Medelexis-Kunden betreffen, sollten in dieser Beschreibung immer wie folgt markiert sein: Proprietär: Exklusiv für Medelexis Kunden

Upgrade

  • Beachten Sie bitte die unten beschriebenen Voraussetzungen
  • Wir empfehlen alles zuerst auf einer Test-Datenbank durchzuspielen und zu testen. Sie könne dafür z.B. einen MySQL Docker Container gebrauchen
  • Falls Sie eine MySQL-Datenbank unter einem Betriebsystem verwenden, dessen Dateiname case-sensitiv ist (MacOSX/Linux), müssen Sie am geeigeneten Ort ein my.cnf mit dem folgenden Inhalt erstellen
  [mysqld]
  lower_case_table_names=1

erstellen

  • Danach müssen wie aus einer der 3 Möglichkeinte Elexis auf 3.4 hochziehen
    • Automatischer Upgrade (nur für Medelexis-Kunden)
    • Elexis OpenSource 3.4.0 neu installieren
    • Elexis Kern und Features via neue P2-Seite aktualisieren

Voraussetzungen

  • Elexis 3.x läuft nicht mit Java 9. Wahrscheinlich mit Java 1.7. Wir empfehlen Java 8 (OpenJDK oder Sun/Oracle).
  • Vorgängig muss eine Elexis Version 3.1.x installiert sein. Upgrade von 2.1.x oder 3.0 läuft wegen in 3.2 neu erstellten User_/Right_-Tabellen.
  • Der MySQL-Server muss mindestens auf der Version 5.7 hochgezogen sein (Nur damit werden die neu gebrauchten SQL-Views unterstützt). Gemäss https://dev.mysql.com/doc/refman/5.7/en/upgrading.html muss man von 5.5 in zwei Schritten (via 5.6) auf 5.7 upgraden. jede andere Methode als mysqldump -> -> restore ist nicht praktikabel, weil bei MySQL (unter Windows...) zwar Upgrades mit gleicher ersten Nachkommastelle direkt vom Installer unterstützt werden - aber solche mit wechselnder erster Nachkommastelle nicht mehr.

Elexis OpenSource 3.4.0 neu installieren

Damit alle gebrauchten Features installier sind, bevor mit dem Upgrade der Datenbank-Tabellen begonnen wird, empfehlen wir wärmstens folgendes Vorgehen

  • DemoDB (mit Administrator) gemäss Installation_Elexis_3.0_demoDB installieren
  • Elexis 3.4 Kern für Ihr Betriebssystem von [Download-Seite herunterladen und entpacken
  • Alle gewünschten Features (zumindest die früher gebrauchten) via Menü "Hilfe..Neue Software installieren" installieren, jeweils nur eine P2-Seite auswählen
  • DemoDB löschen
  • Altes $HOME/elexis/elexis-3.log löschen

Elexis Kern und Features via neue P2-Seite aktualisieren

  • Elexis auf 3.1.9 aktualisieren über "Hilfe..Auf Updates überprüfen"
  • Das Feature "Perspective" (falls vorhanden) deinstallieren via Hilfe..über Elexis".
  • Das Feature "Iatrix KG mit persönlicher Anammnese.." (falls vorhanden) deinstallieren via Hilfe..über Elexis".
    • Knopf "Installation Detail" drücken
    • Im Reiter "Installierte Software" den Punkt "Perspektiven" anwählen und auf Knopf "Uninstall" drücken. Mit "Finish" bestätigen.
    • Neu starten
  • Menü "Hilfe..Neue Software installieren" anwählen. Link "Work with the list of select software site" wählen, dort
    • Eintrag "Elexis 3 Core release" mit "Edit" bearbeiten, Name auf "Elexis 3 Core 3.4" und Location auf "http://download.elexis.info/elexis/3.4/p2/elexis-3-core" setzen. Mit "OK" bestätigen
    • Eintrag "Elexis 3 Base release" mit "Edit" bearbeiten, Name auf "Elexis 3 Base 3.4" und Location auf "http://download.elexis.info/elexis/3.4/p2/elexis-3-base" setzen. Mit "OK" bestätigen
    • Auf "Cancel" drücken
  • Elexis wird jetzt über "Hilfe..Auf Updates überprüfen" auf Elexis auf 3.4 hochgezogen werden

Upgrade Medelexis-Kunden

Der Upgrade wird von Medelexis mit Praxis abgesprochen und sollte danach ohne manuelle Anpassungen bei jeder Arbeitsstation beim nächsten Herunterfahren erfolgen. Bei Prolemen bitte bei Medelexis AG [Kontakt:https://medelexis.ch/?page_id=1061] melen

Arbeiten nach dem Upgrade

Nach dem Upgrade müssen

  • Medelexis-Kunden via "Datei..Stamdaten aktualisieren", den Artikelstamm und Tarmed 1.08 und 1.09 installieren. Das geht lange (Stunden?).
  • OpenSource Anwender müssen
    • Artikelstamm Version 5 von http://artikelstamm.elexis.info runterladen und importieren
    • Tarmed 1.0.8 und 1.0.9 runterladen unde installieren
  • Wechsel auf die neue Tabellenstruktur erzwingen mit
    • Views "Artikel" öffnen, alle Tabs anklicken
    • Views "Leistungen" öffnen, alle Tabs anklicken
    • Views "Lager", "Inbox", "Omnivore" öffnen
    • Patientenperspektive öffnen, einen Patienten auswählen und letzte Konsultation öffnen
  • Gegebenfalls Blöcke angepasst werden, falls Tarmed/Artikel nicht mehr korrekt zugeordnet werden können. Falls dies der Fall ist, wird bei der Ausführung des Makros darauf hingewiesen. Die Landartpraxis Sevelen hat eine Liste von Blöcken für Tarmed 1.09 erstellt. Die Anleitung zum Installeiren dazu finden Sie unter Tarmed_1.09_Blöcke
  • Neues $HOME/elexis/elexis-3.log unter einem sprechenden Namen abspeichern, und damit es bei Problemen mit dem Upgrade analysiert werden kann

Elexis-Server

Der neu entwickelte Elexis Server ist Teil der neuen technischen Grundlagen um Elexis für die Zukunft fit zu halten. Er erlaubt es, Zugriffe auf mehrfach gebrauchte Ressourcen wie Konsultationen, Agenda-Termine sicher zu synchronisieren. Die FHIR-Schnittstelle erlaubt es, Web- (und andere) Applikationen einen sicheren Zugriff auf einzelnen Komponenten von Elexis zu geben.

Er wurde entwickelt, weil Medix in ihren neuen Gruppenpraxen Rockethealth einsetzt. Rockethealth ist eine auf einem Tablet laufende Applikation, um Konsultationen strukturiert zu erfassen.

Dieselbe FHIR-Schnitstelle wird es Elexis erlauben, mit anderen Schweizer eHealth-Applikationen Daten austauschen, z.B. Impdaten, Konsultationen bei Vertretungen, Überweisunen an Spezialisten.

Der Einsatz eines Elexis-Servers erfordert einen gewissen, noch nicht gut dokumentierten, Einrichtungsaufwand. Wer bisher ohne Elexis Server ausgekommen ist, hat (bis jetzt) keinen zwingenden Grund einen Elexis-Server zu gebrauchen. Sein Einsatz kann jedoch sinnvoll sein, wenn bei einer Praxis gehäuft Probleme vorkommen, weil parallel Personen auf gemeinsame Daten zugreifen, oder man den Elexis-Server dazu verwenden will, im Hintergrund Laborwerte einzulesen, sobald diese verfügbar sind.

Technische Details dazu findet man unter:

  • https://github.com/elexis/elexis-server/blob/master/README.md
